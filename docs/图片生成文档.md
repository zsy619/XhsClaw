# 小红书图片生成系统文档

## 概述

本系统完全复刻 Auto-Redbook-Skills 项目的功能，提供完整的小红书笔记图片生成能力，支持将 Markdown 格式的内容渲染为美观的图片，并提供多种主题样式和分页模式供选择。

## 功能特性

### 1. Markdown 转图片

支持将标准 Markdown 格式的文本内容转换为高质量的图片。系统会自动：
- 解析标题和内容段落
- 识别表情符号
- 处理列表项（有序和无序）
- 多种分页模式处理长内容

### 2. 封面生成

提供单独的封面生成功能，支持：
- 自定义主标题
- 自定义副标题
- 多种主题样式
- 可配置的图片尺寸
- 封面与正文一体化设计

### 3. 8种精美主题

系统内置 8 种精美主题，完全复刻原项目：

| 样式 Key | 样式名称 | 描述 |
|---------|---------|------|
| default | 简约灰 | 经典简约灰色主题 |
| playful-geometric | Playful Geometric | 趣味几何主题 |
| neo-brutalism | Neo-Brutalism | 新野兽派主题 |
| botanical | Botanical | 植物主题 |
| professional | Professional | 专业商务主题 |
| retro | Retro | 复古主题 |
| terminal | Terminal | 终端风格主题 |
| sketch | Sketch | 手绘风格主题 |

### 4. 4种分页模式

支持 4 种灵活的分页模式：

| 模式 | 描述 |
|-----|------|
| separator | 按 `---` 分隔符手动分页 |
| auto-fit | 固定尺寸，自动整体缩放内容，避免溢出/大面积留白 |
| auto-split | 根据渲染后高度自动拆分为多张卡片（推荐） |
| dynamic | 根据内容动态调整图片高度 |

### 5. 推荐尺寸

- 默认尺寸：1080×1440px（3:4 比例），完全符合小红书推荐
- Dynamic 模式最大高度：2160px

## API 接口

### 1. 获取可用样式

**接口地址：** `GET /api/xiaohongshu-renderer/styles`

**请求示例：**
```bash
curl -X GET http://localhost:8000/api/xiaohongshu-renderer/styles \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**响应示例：**
```json
{
```

### 2. 渲染 Markdown 为图片

**接口地址：** `POST /api/xiaohongshu-renderer/render`

**请求参数：**
```json
{
  "markdown_content": "✨ 超实用分享！\n\n今天给大家带来超棒的心得～",
  "style_key": "default",
  "output_prefix": "my_note",
  "mode": "auto-split",
  "width": 1080,
  "height": 1440,
  "max_height": 2160
}
```

**参数说明：**
- `markdown_content`: 必填，Markdown 格式的内容
- `style_key`: 选填，样式 key，默认为 "default"
- `output_prefix`: 选填，输出文件名前缀
- `mode`: 选填，分页模式，默认为 "auto-split"
- `width`: 选填，宽度，默认 1080
- `height`: 选填，高度，默认 1440
- `max_height`: 选填，最大高度（dynamic 模式），默认 2160

**请求示例：**
```bash
curl -X POST http://localhost:8000/api/xiaohongshu-renderer/render \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "markdown_content": "✨ 超实用分享！\n\n今天给大家带来超棒的心得～\n\n📝 首先要注意以下几点：\n- 第一点真的很重要\n- 第二点也不能忽略",
    "style_key": "playful-geometric",
    "output_prefix": "test_note",
    "mode": "auto-split"
  }'
```

**响应示例：**
```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "success": true,
    "message": "渲染成功",
    "images": [
      "/xiaohongshu-renderer/image/test_note_20240101120000.png",
      "/xiaohongshu-renderer/image/test_note_20240101120000_0.png"
    ]
  }
}
```

### 3. 生成封面

**接口地址：** `POST /api/xiaohongshu-renderer/cover`

**请求参数：**
```json
{

  "subtitle": "让你的生活更美好",
  "style_key": "default",
  "output_prefix": "cover",

```

**参数说明：**
- `title`: 必填，主标题
- `subtitle`: 选填，副标题
- `style_key`: 选填，样式 key，默认为 "default"
- `output_prefix`: 选填，输出文件名前缀
- `width`: 选填，宽度，默认 1080
- `height`: 选填，高度，默认 1440

**请求示例：**
```bash
curl -X POST http://localhost:8000/api/xiaohongshu-renderer/cover \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "title": "超实用分享",
```

**响应示例：**
```json

  "data": {
    "success": true,
    "message": "封面生成成功",
```

### 4. 获取生成的图片

**接口地址：** `GET /api/xiaohongshu-renderer/image/{filename}`

直接访问该 URL 即可获取图片文件。

## Markdown 格式说明

系统支持以下 Markdown 格式：

### 标题
第一行文本会被自动识别为标题。

### 列表
- 无序列表：使用 `-` 或 `•` 开头
- 有序列表：使用数字加 `.` 或 `、` 开头

### 分页分隔符（Separator 模式）
使用 `---` 进行手动分页。

### 表情符号
系统会自动识别和保留表情符号。

### 示例内容

#### Auto-split 模式（推荐）
```markdown
✨ 超实用分享！

今天给大家带来超棒的心得～

📝 首先要注意以下几点：
- 第一点真的很重要
- 第二点也不能忽略
- 第三点是关键

💡 最后给大家一些小建议：
1. 建议1
2. 建议2
3. 建议3

希望对大家有帮助哦～
```

#### Separator 模式（手动分页）
```markdown
✨ 第一页内容

这是第一页的内容。

---

✨ 第二页内容

这是第二页的内容。

---

✨ 第三页内容

这是第三页的内容。
```

## 核心服务类

### RendererService

位置：`internal/service/renderer.go`

主要方法：

#### NewRendererService()
创建渲染服务实例。

```go
service := service.NewRendererService()
```

#### GetStyles() []StyleConfig
获取所有可用样式配置。

```go
styles := service.GetStyles()
```

#### GetStyle(key string) StyleConfig
获取指定样式配置。

```go
style := service.GetStyle("playful-geometric")
```

#### RenderMarkdownToImage(markdown, styleKey, outputPrefix string, mode PaginationMode, width, height, maxHeight int) ([]string, error)
将 Markdown 内容渲染为图片。

参数：
- `markdown`: Markdown 内容
- `styleKey`: 样式 key
- `outputPrefix`: 输出文件名前缀
- `mode`: 分页模式
- `width`: 宽度
- `height`: 高度
- `maxHeight`: 最大高度（dynamic 模式）

返回：
- 图片 URL 数组
- 错误信息

#### GenerateCoverOnly(title, subtitle, styleKey, outputPrefix string, width, height int) (string, error)
单独生成封面图片。

参数：
- `title`: 主标题
- `subtitle`: 副标题
- `styleKey`: 样式 key
- `outputPrefix`: 输出文件名前缀
- `width`: 宽度
- `height`: 高度

返回：
- 图片 URL
- 错误信息

## 样式配置

每个样式包含以下配置：

```go
type StyleConfig struct {
	Key           string        // 样式 key
	Name          string        // 样式名称
	Primary       color.Color   // 主色
	Secondary     color.Color   // 辅助色
	Background    color.Color   // 外层背景色（card-container）
	CardInner     color.Color   // 内层卡片背景色（card-inner）
	TextPrimary   color.Color   // 主文本色
	TextSecondary color.Color   // 辅助文本色
	Accent        color.Color   // 强调色
}
```

## 统一卡片结构

系统采用统一的三层卡片结构：

1. **card-container**：外层容器，浅灰背景
2. **card-inner**：内层卡片，主题背景
3. **card-content**：内容排版层

## 文件存储

生成的图片存储在：
- 目录：`./public/images/`
- 文件名格式：`{prefix}_{timestamp}.png`（封面）或 `{prefix}_{timestamp}_{index}.png`（正文卡片）

## 使用示例

### Go 代码示例

```go
package main

import (
	"xiaohongshu/internal/service"
)

func main() {
	// 创建渲染服务
	renderer := service.NewRendererService()

	// 示例 1：生成封面（Terminal 主题）
	coverURL, err := renderer.GenerateCoverOnly(
		"超实用分享",
		"让你的生活更美好",
		"terminal",
		"my_cover",
		1080,
		1440,
	)
	if err != nil {
		panic(err)
	}
	println("封面URL:", coverURL)

	// 示例 2：渲染 Markdown（Playful Geometric 主题，Auto-split 模式）
	markdown := `✨ 超实用分享！

今天给大家带来超棒的心得～`

	imageURLs, err := renderer.RenderMarkdownToImage(
		markdown,
		"playful-geometric",
		"my_note",
		service.PaginationAutoSplit,
		1080,
		1440,
		2160,
	)
	if err != nil {
		panic(err)
	}
	println("生成的图片:", imageURLs)

	// 示例 3：使用 Dynamic 模式
	dynamicImages, err := renderer.RenderMarkdownToImage(
		markdown,
		"botanical",
		"dynamic_note",
		service.PaginationDynamic,
		1080,
		1440,
		2160,
	)
	if err != nil {
		panic(err)
	}
	println("动态模式图片:", dynamicImages)
}
```

## 注意事项

1. **字体依赖**：系统依赖系统字体文件，确保系统安装了中文字体（如 PingFang）
2. **图片尺寸**：推荐使用 1080×1440 作为标准尺寸，符合小红书 3:4 比例要求
3. **分页模式选择**：
   - 内容长短难控时，推荐使用 **auto-split** 模式
   - 需要精确控制分页时，使用 **separator** 模式
   - 希望单张图片展示时，使用 **auto-fit** 或 **dynamic** 模式
4. **文件权限**：确保 `./public/images/` 目录有写入权限
5. **主题一致性**：封面背景、标题渐变和正文卡片背景会按主题自动匹配
